# Interview Questions: Model Tracking

## Beginner Questions
1. **What is experiment tracking and why is it necessary?**
   - *Answer concept:* It is the process of saving parameters, metrics, and models for every training run. It is necessary to ensure reproducibility, allow comparison between different approaches, and maintain a history of model development.
2. **What is the difference between a parameter and a metric in MLflow?**
   - *Answer concept:* A parameter is an input setting chosen before training begins (e.g., learning rate). A metric is a calculated output evaluated during or after training (e.g., accuracy, loss).
3. **What is an artifact in the context of model tracking?**
   - *Answer concept:* An artifact is an output file generated by a run. The most important artifact is the serialized model itself, but it can also include plots (e.g., ROC curve images) or data files.

## Conceptual Questions
4. **Explain the concept of a Model Registry.**
   - *Answer concept:* A model registry is a centralized repository that manages the lifecycle of ML models. It allows teams to transition models through stages like "Staging", "Production", and "Archived", ensuring everyone knows which model version is currently serving traffic.
5. **If you run `mlflow.autolog()`, what are the trade-offs compared to manual logging?**
   - *Answer concept:* Autologging is very fast to set up and ensures you don't forget to log standard metrics. However, it can log too much information, cluttering the UI, and it might not capture custom metrics specific to your business logic that manual logging would allow.

## Practical Questions
6. **You have trained 50 different models and logged them with MLflow. How do you find the best one?**
   - *Answer concept:* I would open the MLflow UI, navigate to the experiment, and sort the runs table by the primary evaluation metric (e.g., descending order for accuracy, ascending for MSE).
7. **How does experiment tracking help in a team environment?**
   - *Answer concept:* By using a centralized tracking server, team members can see what experiments others have run. This prevents duplicated effort (e.g., tuning the exact same hyperparameters) and allows for easier collaboration and review of results.

## Output Interpretation
8. **In the MLflow UI, you are looking at a parallel coordinates plot. What does it help you understand?**
   - *Answer concept:* A parallel coordinates plot connects parameters to metrics across many runs. It helps visualize trends, such as "whenever max_depth is high and learning_rate is low, the accuracy tends to increase."
